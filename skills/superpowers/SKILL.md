# Superpowers

Complete dev methodology. Stops random coding. Forces planning, testing, review.

## Commands & Workflows
- `/brainstorming`: Tease out spec. Validate design in chunks. No raw coding.
- `/writing-plans`: Break approved design into 2-5 min tasks. Exact files, complete code, verification steps.
- `/executing-plans`: Dispatch subagents per task. Two-stage review (spec compliance, code quality).
- `/test-driven-development`: RED-GREEN-REFACTOR. Write test -> fail -> write code -> pass. Delete code written before tests.
- `/systematic-debugging`: Root-cause analysis. Stop guessing. 4-phase RCA process.
- `/using-git-worktrees`: Isolate workspace on new branch. Clean test baseline.
- `/requesting-code-review`: Review against plan. Report by severity. Block on critical.
- `/finishing-a-development-branch`: Verify tests, present merge/PR options, clean worktree.

## Core Directives
1. ALWAYS ask for intent before coding.
2. ALWAYS generate test before implementation (TDD).
3. ALWAYS root-cause bugs systematically.
